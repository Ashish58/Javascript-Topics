Data Types in JavaScript

Back to Topics

Description:

JavaScript has several data types categorized into primitive and non-primitive types:

Primitive Data Types:

  • String: Represents textual data.
  • Number: Represents numeric data (both integers and floating-point numbers).
  • Boolean: Represents true or false values.
  • Undefined: A variable that has been declared but not assigned a value.
  • Null: Represents the intentional absence of any object value.
  • Symbol: Represents a unique and immutable value, often used as object property keys.
  • BigInt: Represents integers larger than the Number type can handle.

Non-Primitive Data Types (Reference Types):

  • Object: A collection of key-value pairs, including arrays, functions, and more complex objects.

Example:

// Primitive Data Types
let name = "Alice";               // String
let age = 30;                     // Number
let isMember = false;             // Boolean
let x;                            // Undefined
let y = null;                     // Null
let id = Symbol("id");            // Symbol
let bigNum = 9007199254740991n;   // BigInt

// Non-Primitive Data Type
let person = {                    // Object
    name: "Alice",
    age: 30
};

Primitive vs Non-Primitive Data Types:

  • Primitive Data Types are immutable and compared by their value.
  • Non-Primitive Data Types (objects) are mutable and compared by their reference.

Practice Question:

Declare variables of each primitive type and log their types using typeof.

Expected Output:

console.log(typeof name);   // "string"
console.log(typeof age);    // "number"
console.log(typeof isMember); // "boolean"
console.log(typeof x);      // "undefined"
console.log(typeof y);      // "object" (note: null is technically an object)
console.log(typeof id);     // "symbol"
console.log(typeof bigNum); // "bigint"
